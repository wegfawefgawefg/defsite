Shadowing gets a bad reputation when it is invisible. In this model, definitions are explicit tags in the markup, so when a symbol changes behavior, you can see exactly where it happened.

The resolver strategy is simple: start at current scope, then walk parents until root. No import graph magic, no hidden implicit precedence layers.

Keep broad reusable symbols generic, and use longer names for local overrides. Readability improves immediately.

Team conventions that helped

  • Prefer `def-post-card` globally, `def-post-card-compact` for local variants.
  • Document the scope boundary where shadowing is intentional.
  • Add fixture tests for each shadowing behavior you depend on.
