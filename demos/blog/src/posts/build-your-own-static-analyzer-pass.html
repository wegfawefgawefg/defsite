General-purpose linters are good at universal rules, but every codebase has local invariants they cannot know. A custom static analysis pass lets you encode those invariants directly: forbidden call paths, layering violations, unchecked return codes, or unsafe API combinations.

I start with one high-value rule and make it accurate before adding more. A noisy analyzer loses trust instantly. A precise analyzer with one rule saves incidents and earns adoption quickly.

Implementation strategy

  1. Define the invariant with concrete code examples.
  2. Choose an AST or IR stage where the signal is clear.
  3. Emit diagnostics with actionable remediation text.
  4. Roll out in warning mode, then enforce once false positives are gone.
Publish analyzer findings in CI with links to rule docs and migration examples. Developers accept strict checks when remediation is clear.

Once a pass stabilizes, it becomes institutional memory in executable form. New teammates inherit policy through tooling rather than tribal knowledge.
